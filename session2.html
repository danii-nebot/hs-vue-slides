### List rendering The **v-for** directive ``` <li v-for="(item, index) in itemsArray :key="item.id"> {{ variableInParentScope }} {{ index }} - {{ item.anyProperty }} </li> ``` *index* is optional [It's recommended](https://vuejs.org/style-guide/rules-essential.html#use-keyed-v-for) to always provide a **:key** attribute with v-for for performance reasons
#### v-for variants: JS Objects It's possible to iterate through object properties, for example: ``` const article = { title: 'How to do lists in Vue', author: 'Jane Doe', publishedAt: '2016-04-10' } ``` ``` <li v-for="(value, key) in article"> {{ key }}: {{ value }} </li> ``` Just like *index*, *key* here is optional
#### v-for variants: range Repeat the template X times ``` <li v-for="n in 7"> {{ getDwarfNameFromIndex(n) }} </li> ``` **gotcha**: the list will begin at 1, not zero!
#### List rendering /3 ``` <ul> <template v-for="item in itemList"> <li>{{ item }}</li> <li class="divider" role="presentation"></li> </template> </ul> ``` Just like v-if, you can also use a *template* tag with v-for to render a block of multiple elements if needed
#### List rendering /4 **Caveat**: [Avoid v-if and v-for](https://vuejs.org/style-guide/rules-essential.html#avoid-v-if-with-v-for) on the same element: ``` // Avoid! <ul> <li v-for="user in users" v-if="user.isActive" :key="user.id" > {{ user.name }} </li> </ul> ```

#### Form input bindings Vue uses the **v-model** directive to go from the cumbersome: ``` <input :value="text" @input="event => text = event.target.value"> ``` to the much nicer and concise: ``` <input v-model="text"> ```
#### Form input bindings /2 v-model can be used on inputs of different types, *textarea*, and *select* elements ``` <!-- bad --> <textarea>{{ text }}</textarea> <!-- good --> <textarea v-model="text"></textarea> ```
#### Form input bindings /3 Form input binding are extremely flexible, and powerful, but also can be confusing. When in doubt, always check [the Vue 3 docs](https://vuejs.org/guide/essentials/forms.html)
