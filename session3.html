### Functional Programming (FP) **Pure functions** must: - Always return the same return value for the same arguments - Be Side-effect-free ``` function multiply(a, b) { return a * b; } ```
#### Why is FP important? - Easier to follow, debug and test - Performance boost!
#### What is not FP? ``` // Loops: for, while, for...in, ... // Variable declarations with var or let // Object mutation: cat.color = 'orange'; // Array mutation: .push(), .pop(), .sort(), .splice()... // Void functions (that don't return a value) ```
#### FP in JavaScript: Array Iterators - Accept another function as a parameter (Higher order functions) - Iterate on the array elements, invoking the function ``` // This is a very common pattern in modern JS: return listOfElements .filter(item => ...) // to filter out unwanted elements .map(item => ...) // to change the structure of the data .reduce((acc, item) => ...); // does literally anything // the best part is, after all the data manipulation, // the array listOfElements is completely unchanged! ```

### FP in Vue: Computed properties Avoid clutter in the template Avoid code repetition Can be cached for performance gains
#### computed() Change this: ``` <p>Has published books:</p> <span>{{ author.books.length > 0 ? 'Yes' : 'No' }}</span> ``` Into this: ``` // a computed ref const publishedBooksMessage = computed(() => { return author.books.length > 0 ? 'Yes' : 'No' }) </script> <template> <p>Has published books:</p> <span>{{ publishedBooksMessage }}</span> </template> ```

### Watchers FP is cool But sometimes we NEED side effects... We can use the [watch()](https://vuejs.org/guide/essentials/watchers.html#basic-example) function: ``` watch(value, (newValue, oldValue) => { // perform some side effects // when value changes }) ```
#### Template Refs Vue abstracts us from the DOM but... Sometimes we NEED access to the DOM elements. For this, we use the special attribute [ref](https://vuejs.org/guide/essentials/template-refs.html) ``` <input ref="input"> ``` (yes, it should look familiar)
