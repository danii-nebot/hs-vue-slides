### Component Driven Development (CDD) Roughly means designing your application by building loosely-coupled independent units: **The components** This allows for faster, more scalable, and efficient development
#### FIRST principles by [Addy Osmani](https://addyosmani.com/first/) - Keep it (F)ocused - Keep it (I)ndependent - Keep it (R)eusable - Keep it (S)mall - Keep it (T)estable

#### Passing Props Props are custom attributes you can register on a component Using props, information flows from the parent to the child components ``` <script setup> defineProps(['color']) </script> ```
defineProps({ title: String, likes: Number })
#### Danger: Prop drilling Data needs to go through many levels to reach its destination ![](https://media.geeksforgeeks.org/wp-content/uploads/20210618101141/Untitled.png)
#### Props are a One-Way Data Flow Vue will warn you if you mutate a prop inside a child component ``` const props = defineProps(['hasEyesOpen']) props.hasEyesOpen = true // ‚ùå [Vue warn] Set operation on key "hasEyesOpen" failed: target is readonly.[object Object] ```
#### Usecase number 1: The prop is used to pass in an initial value Solution: define a local data property that uses the prop as its initial value: ``` const props = defineProps(['initialCounter']) // counter only uses props.initialCounter as the initial value; // it is disconnected from future prop updates. const counter = ref(props.initialCounter) ```
#### Usecase number 2: The prop needs to be processed Solution: define a computed property using the prop's value ``` const props = defineProps([{ color: String, }]) // computed property that auto-updates when the prop changes const colorStyle = computed(() => `background-${color.toLowerCase()}`) ```
#### Listening to Events Using events, information flows from the child components to the parent ``` <button @click="$emit('custom-event')">Cliek me!</button> ``` And... We already know how to listen to events! üôå
#### Component Slots It's often useful to be able to pass content to a component We use the <slot> tag as a "placeholder" where we want the content to go... and that's it. We're done!
