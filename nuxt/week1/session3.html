Nuxt Routing

### Routes deep dive
#### The pages/ directory Nuxt routing is based on **vue-router** and generates the routes from every component created in the pages/ directory, based on their filename.
#### Example Structure ``` pages/ --| index.vue --| about.vue --| posts/ ----| [id].vue ```
#### Dynamic Routes Any page name within square brackets will be turned into a dynamic route parameter Mix and match of multiple parameters and even non-dynamic text can be done within a file name or directory
#### Dynamic Routes Example ``` -| pages/ ---| index.vue ---| users-[group]/ -----| [id].vue ```
#### Optional router parametrs If you want a parameter to be optional, you must enclose it in double square brackets - for example, `~/pages/[[slug]]/index.vue` and `~/pages/[[slug]].vue` will match both / and /test.
#### Catch-all Route A "Catch-all" route can be created by using a file named like `[...slug].vue`. This will match all routes under that path.
#### Accessing route parameters We can use the `$route` object in the template: ``` <template> <p>{{ $route.params.group }} - {{ $route.params.id }}</p> </template> ```
#### Accessing route parameters /2 Or alternatively, the global `useRoute` function from our script setup: ``` const route = useRoute() console.log(route.params.id) ```
#### Nested Routes It is possible to display nested routes using the following structure: ``` -| pages/ ---| parent/ ------| child.vue ---| parent.vue ```
#### Nested Routes /2 To display the child.vue component, you have to insert the **<NuxtPage>** component inside pages/parent.vue: ``` <template> <div> <h1>I am the parent view</h1> <NuxtPage /> </div> </template> ```

Navigation

Nuxt uses the <NuxtLink> Component to link pages, instead of <router-link>

The magic: When a <NuxtLink> enters the viewport on the client side, Nuxt will automatically prefetch components and payload (generated pages) of the linked pages ahead of time, resulting in faster navigation.

### Transitions
#### Adding Transitions Transitions must be created using CSS classes with the `transition` property They need to be added with the global styles of the app, for example in `~assets/css/global.css` or in `app.vue`
#### Example CSS transition ``` .page-enter-active, .page-leave-active { transition: all 0.4s; } .page-enter-from, .page-leave-to { opacity: 0; filter: blur(1rem); } ```
#### Default Transitions Transitions can be enabled in nuxt.config.ts to apply an automatic transition for all your pages or layout changes: ``` // in nuxt.config.ts export default defineNuxtConfig({ app: { pageTransition: { name: 'page', mode: 'out-in' }, layoutTransition: { name: 'layout', mode: 'out-in' } }, }) ```
#### Custom Per Page Transitions To set a different transition for a page, we can use the **pageTransition** key in definePageMeta of the page: ``` definePageMeta({ pageTransition: { name: 'custom' } }) ```

### SEO and Meta
#### SEO 101 Search engine optimization (SEO) is the process of improving the quality and quantity of website traffic to a website or a web page from search engines. [Google SEO Fundamentals](https://developers.google.com/search/docs/fundamentals/seo-starter-guide)
#### Global Meta values The `app.head` property in `nuxt.config.ts` allows to customize the head for the entire app. ``` export default defineNuxtConfig({ app: { head: { title: 'My App', meta: [ // { name: 'description', content: 'My amazing site.' } ], } } }) ```
#### Default Meta values Out-of-the-box, Nuxt provides the following sane defaults, which you can override if needed: ``` charset: utf-8 viewport: width=device-width, initial-scale=1 ```
#### Custom Meta Per Page Nuxt provides Title, Base, NoScript, Style, Meta, Link, and Head components so that you can interact directly with your metadata within your component's template. Because these component names match native HTML elements, they must be capitalized in the template.
#### Custom Meta Per Page Example ``` <script setup> const title = ref('My Product Name') </script> <template> <div> <Head> <Title>{{ title }}</Title> <Meta name="description" :content="title" /> </Head> <h1>{{ title }}</h1> </div> </template> ```
