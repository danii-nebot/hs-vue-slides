### The Views Nuxt provides several layers to implement the user interface of your application. Just as with a "regular" Vue app, the entry point of the application is **app.vue**, which will in turn render every page.
#### app.vue ![](https://nuxt.com/assets/docs/getting-started/views/app.svg)
#### Components ![](https://nuxt.com/assets/docs/getting-started/views/components.svg)
#### Components Single page components are reusable pieces of the user interface Common components go into the **components/** directory, and they will be automatically available across the application
#### Nuxt Components exercise - Create a component named "AlertBox". This component should render its content in a default **slot**. - Add a "type" prop to the component of type string - Add styles to the component according to its type (red, orange, blue, green color respectively) - Bonus: Use TS to limit the type to one of these four values: error, warning, info, success
#### Pages ![](https://nuxt.com/assets/docs/getting-started/views/pages.svg)
#### Pages Pages represent views use for each specific **route pattern**. Every file in the **pages** directory represents a different route displaying its content.
#### Nuxt Pages exercise - Delete app.vue and create a new page named "index". - Create a new page named "about". - Add a component named "AboutText", with Lorem ipsum text. (This component will be used only in the "/about" page) - How can we navigate to this page?
#### Layouts ![](https://nuxt.com/assets/docs/getting-started/views/layouts.svg)
#### app.vue If we have a single layout for our application, the simplest way is to use the template of *app.vue* In case we want more flexible and powerful options, we can create **Nuxt layouts**
#### Layouts Layouts are wrappers around pages that contain a common User Interface for several pages, such as a header and footer display. Layouts are Vue files using *slot* components to display the page content, and reside in the **layouts** directory
#### Layouts /2 We can override the layout of a page by using *definePageMeta*: ``` definePageMeta({ layout: "my-layout", }); ```
#### Nuxt Layouts exercise - Create a default layout - Add two components: DefaultHeader and DefaultFooter (They should render above and below the page content, respectively) - Create a new layout without a footer, and set your *about page* to use it.
### Assets Nuxt uses two directories to handle assets like stylesheets, fonts or images. - The **public/** directory content is served at the server root as-is. - The **assets/** directory contains by convention every asset that you want the build tool to process.
#### public/ Directory It is used for **static assets** publicly available at a defined URL of your application. Assets can be accessed from the **root of the URL**
#### assets/ Directory - It is a **convention** - there is no auto-scan in this case - It is used for other kind of assets that can be preprocessed for performance or caching purposes such as stylesheets, fonts or SVG - In your application's code, you can reference a file located in the assets/ directory by using the **~/assets/** path.

Tailwindcss

Rapidly build modern websites without ever leaving your HTML.

A utility-first CSS framework packed with classes like flex, pt-4, text-center and rotate-90 that can be composed to build any design, directly in your markup.
#### What are utility classes? Utilities are simple HTML classes typically scoped to a **single CSS property**, like border-style or background-color. Utilities can be used additively to style an object from scratch, or to override a style defined in component CSS. Source: [U.S. Web Design System (USWDS)](https://designsystem.digital.gov/utilities/)
#### Advantages of a utility-first approach - Your CSS stops growing. - Making changes feels safer. CSS is global and you never know what you're breaking when you make a change. - Forget about needing to use **!important** or specificity woes! - No more wasting energy inventing class names (BEM)
#### How does it look like? ``` <div class="p-6 max-w-sm mx-auto bg-white rounded-xl shadow-lg flex items-center space-x-4"> <div class="shrink-0"> <img class="h-12 w-12" src="/img/logo.svg" alt="ChitChat Logo"> </div> <div> <div class="text-xl font-medium text-black">ChitChat</div> <p class="text-slate-500">You have a new message!</p> </div> </div> ```
#### Why not just use inline styles? In some way, we are doing just that, but with added benefits: - Designing with constraints (Design System) - Tailwind provides **responsive utilities** - Hover, focus, and other state variants
#### Nuxt + Tailwind exercise - Follow the [instructions here](https://tailwindcss.com/docs/guides/nuxtjs#3) to add Tailwind CSS to your Nuxt project - Get familiar with Tailwind CSS by adding styles to your component, pages and layouts
