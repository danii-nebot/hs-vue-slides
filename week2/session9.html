Vue Router

### Vue Router [Official Router library for Vue 3](https://router.vuejs.org/) However, it's not part of the core - must be installed additionally ``` npm i vue-router@4 ``` Vue Router allows us to navigate between the different pages in our SPA (Single App Application)
### Server-Side - The client is making a request to the server on every URL change - "Traditional" website
.
.
### Client Side: - The routing happens in the browser itself using JavaScript - Vue Router simply updates what part of the app that is currently being displayed - **Single Page Application (SPA)**
### How to the Vue Router We will map our **components** to **routes**, and configure Vue Router know where to render them
#### How Vue Router is configured By convention, we use a **router** folder to hold the configuration of the router And a **views** folder to hold the Views component (our pages)

Remember this?

Example configuration (in router/index.js) ``` const routes = [ { path: '/', name: 'Home', component: Home }, { path: '/about', name: 'About', // etc ... } ] ```
The **path** indicates the actual route, in terms of the URL, that the user will be taken to The **name** allows us to give this route a name so we can use that name throughout our application to refer to this route The **component** allows us to specify which Vue Component to render at that route
#### The router-view Component The component matched by the route will render here We can think of it as the "slot" of the route ``` <router-view></router-view> ```
#### The router-link component Specify the link by passing the `to` prop The **router-link** will render an **a** tag with the correct href attribute: ``` <router-link to="/">Go to Home</router-link> <router-link to="/about">Go to About</router-link> ```
#### Programmatic Navigation: push! From the template, Vue Router is exposed as **$router** ``` <button @click="$router.push('about')">About page</button> ``` From the script setup: ``` import { useRouter, useRoute } from 'vue-router' const router = useRouter() const route = useRoute() ```
#### Dynamic Routes A **param** is denoted by a colon **:** With the flag *props: true* the route params will be passed to the Component as a prop! ``` const routes = [ // dynamic segments start with a colon { path: '/users/:id', component: User, props: true }, ] ```
#### Catch all / 404 Not found Route Thanks to the power of **regexp** we can create a custom param that will match anything ``` const routes = [ // will match everything and put it under `$route.params.pathMatch` { path: '/:pathMatch(.*)*', name: 'NotFound', component: NotFound }, ```
