Asynchronous JS

### Synchronous Programming The JS engine executes the code *line by line* This is called *Blocking Code* because one line needs to finish executing before we jump to the next ones
### Aynchronous Programming Performs *Non blocking operations* The JS engine can move to another "task" before the previous finishes, and deal with multiple requests simultaneously ``` console.log('begin'); setTimeout(() => { console.log('lol'); }, 2000); console.log('end'); ```
#### Callbacks We can use *higher order functions* to set up handlers for Asynchronous operations ``` function getGithubUser(id, callback) { console.log("Fetching user from GH API..."); setTimeout(() => { callback({id: id, username: 'danii-nebot'}); }, 1000); } getGithubUser(1, (user) => { console.log("GH username:", user.username); }); ```
#### Callbacks can be chained To deal with multiple async operations that depend on each other ``` getGithubUser(1, (user) => { getRepositories(user.username, (repos) => { // ... and so on ```

### Async / Await It's just *syntantic sugar* of using promises What this means is, it's **still using promises internally**, but even nicer code!
#### How does it look like? ``` // async / await approach async function displayCommits() { try { const user = await getUser(1); const repos = await getRepositories(user.githubUsername); const commits = await getCommits(repos[0]); // ... } catch (err) { // process err } } displayCommit(); ```
### The Fetch api As always, [check the MDN reference page](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API) and [tutorial](https://developer.mozilla.org/en-US/docs/Web/API/Fetch_API/Using_Fetch) ``` const res = await fetch('http://example.com/movies.json') if (!res.ok) { // handle error } else { const data = (await res.json()) } ```
### Vue Async Components Components with a *top-level await* are automatically considered to have an **async setup**
#### Suspense Suspense is a **built-in component** for orchestrating async dependencies It can render a loading state while waiting for multiple nested async dependencies
